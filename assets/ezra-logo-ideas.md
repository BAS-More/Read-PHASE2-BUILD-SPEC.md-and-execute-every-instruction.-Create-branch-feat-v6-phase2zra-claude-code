# EZRA Logo Directions

The top badge has been removed from the live logo. These are three tighter directions that fit Ezra *and* the software:

## 1. Scribe + Code

![Scribe and Code](./ezra-logo-concept-scribe-code.svg)

Why it fits:

- Keeps Ezra unmistakably as a scribe with the scroll, quill, and Hebrew name.
- Adds software identity through the bracket frame, which reads like code without feeling too generic.
- Best option if you want the clearest balance between tradition and modern tooling.

## 2. Keeper of Standards

![Keeper of Standards](./ezra-logo-concept-standards-tablets.svg)

Why it fits:

- Pushes the governance and standards side harder.
- Feels more authoritative and institutional, which suits ADRs, guard rails, and enforcement.
- Best option if you want EZRA to feel like the steward of rules and architecture.

## 3. Decision Network

![Decision Network](./ezra-logo-concept-decision-nodes.svg)

Why it fits:

- Connects Ezra's role as organizer and teacher to software architecture and decisions.
- The connected nodes hint at ADRs, multi-agent workflows, and governance graphs.
- Best option if you want the brand to lean more technical and product-like.

## My Recommendation

`Scribe + Code` is the strongest fit for the current product. It still feels like Ezra HaSofer, but it also says "software system" immediately.
